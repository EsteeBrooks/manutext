1.What was the use case or problem you were trying to solve with your project?

Some helpful considerations: What is the functional problem and how do you measure it? Who would benefit from solving this problem?

Oftentimes, when I try to read manuscripts of old Hebrew texts, such as old Torahs or Talmudic texts, I find it hard to figure out the letters and piece together the words due to the complicated way the characters are written. The functional problem I was trying to solve was humans having trouble making out characters. 

People who study manuscripts may benefit from this project because they could get a glimpse of the text in a manuscript without having to parse through each individual script character. This could greatly speed up their projects and work. As well, normal people, like me, who may just be reading a manuscript for research purposes or for fun may benefit from this algorithm as it will allow them to easily access the text written in a manuscript. 

2.What type of model did you build and how did you choose this model? Some helpful considerations:Is it a supervised or unsupervised model? If supervised, is it a regression or classification model? Does the problem require an explainable model?

The model I used was Tesseract OCR (optical character recognition), which is an open source model that recognizes text from images. The model has been around for quite some time; originally it was developed by Hewlett-Packard Laboratories Bristols in Colorado between 1985 to 1994. In 2005, it became an open source project by HP and in 2006 until 2018 Google helped develop it. Tesseract works with over 100 different languages.

How the underlying model works is not clear to the outsider who uses Tesseract as one simply imports the model. However, the model can be trained further with one’s own data. I do know that this model does NOT use deep learning for the base model.  

I chose this model firstly because it was free. Although there are other OCR models out there, such as one by Amazon and Google which perform much better than Tesseract, they cost a lot of money and were not in the budget for this project. Additionally, I choose Tesseract over these other models because it supports text recognition for the Hebrew language. This is unlike, for example, Google’s OCR, which does not support Hebrew.  

This model does not need to be explainable to the user (evidently from the fact that even I, as the creator of the program, do not know how the model works). The user just needs to be able to send data through the model in order to get an output of Hebrew text. 

3.What is the ”product” you are building with the results of your model? How does this help an interested persona solve their problem?

The product I am building is a flask website where one can upload an image from the Biblla_Dataset and get the text characters that Tesseract predict is in this file. As well, I output the correct text from the Biblla_Dataset by extracting it from the XML files. 

This helps an interested person solve their problem of not being able to read the text of a manuscript from the Biblla_Dataset because they can upload the image of the manuscript and read it through simple text characters. 

4.What were the data used during the exercise (for modelling, exploratory analysis etc).What are the biases and/or other limitations of this data?

The data I used was from BibllA - An open annotated dataset.

The dataset includes data from Ashkenz, Sephardi and Italian manuscripts making it come from all different sources. The dataset was great for what I was looking for - it had lots of images, which could be used for training, with the correct labels of the text inside each image. (However, in the end I didn’t need to train the model.)

I used this data as well when computing the average accuracy of the model on the entire dataset. As well, I created a histogram of the different accuracy scores. This allowed me to explore how well the model did overall and if it did better in some cases or just performed basically the same overall. 

5.How did you train your model?

The Tesseract model does not need to be trained! One simply has to import the model and use it on an image with the specified language imported. 

6.How did you evaluate the performance of your model? Why did you choose the metrics you chose?

I evaluated the model by using the correct labels for images from the BibllA dataset against the predicted labels from Tesseract. I put each word into an array for each respective string of characters and then computed the Accuracy Score from Scikit-Learn. Using the Accuracy Score makes sense in my case this computes the fraction of correctly classified samples, which in my case is the correct word. 

7.What are the limitations of your product/model/analysis?

The limitations of this product is that the model does not have a high accuracy. Based on the overall average of accuracy in the data analysis , the images overall got a score of 0.0008799837437847445 which is super low. Also, looking at the text predicted when you upload an image is not very helpful and doesn’t seem to make any real Hebrew words. 

8.If you could have had another dataset of your choice, what would that data have been?

I thought my dataset was really good but I felt I could not leverage it completely with Tesseract. Therefore, instead of a different dataset, I might look for a different model. If I had unlimited money I would find another OCR model that has Hebrew and performs better than Tesseract. 

9.If you had more time, what would you do next to improve the project?

If I had more time to improve this project I would have used the BibllA dataset to further train the Tesseract model. This takes some time to do so I could not implement it but it would be interesting to see if I could improve the accuracy score of the model by doing so. 

10.What challenges did you run into during the course of your project?

The most challenging part of this project was using Flask. I found it combersumb to keep reloading the browser (even in debug mode) and upload an image to see if the model was working correctly, as well seeing if the website worked correctly.
